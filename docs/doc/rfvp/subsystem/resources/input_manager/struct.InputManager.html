pub fn peek_control_pulse(&self) -> bool

Return whether a ControlPulse is pending, without consuming it.

This is used when the engine needs to observe the pulse in multiple phases of a frame (e.g., pre-VM motion update and post-VM text reveal), while ensuring the pulse is still consumed exactly once.

Source

pub fn take_control_pulse(&mut self) -> bool

Consume the one-frame ControlPulse flag.

In the original engine, ControlPulse sets a scene flag that is checked during the next frame update and then cleared immediately. That makes it a pulse, not a persistent mode toggle.
